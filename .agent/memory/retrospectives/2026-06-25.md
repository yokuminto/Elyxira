## 四、经验总结

### 模块设计
- **存储抽象层价值大**：breakStorage 对外 API 不变（getNotes/setNote/mergeNotes），内部从 localStorage 切 IndexedDB 对调用方完全透明
- **逻辑分组提交**：20+ 个小提交合并为 3 个逻辑提交（feature + docs + fix），历史清晰可回溯

### 常见陷阱
- **Vue watcher 只追踪引用变化**：`_syncNotesToQuestion` 原地改 `q.notes` 不触发 watcher，需返回 `{ ...q, notes }` 新对象
- **`_drawFromPool` 调用时机**：必须在 `allQuestions` 赋值之后，否则 `_getQuestionById` 全返回 null 吞空池
- **ThinkFilter.flush() 吞最后 6 字符**：因为 flush 调了 feed()，而 feed 默认保留最后 6 字符防 `</think>` 截断

### 可复用模式
- **批量生成 + 进度环**：`Promise.allSettled` + try-catch + 计数回调（其他批量操作可复用）
- **localStorage → IndexedDB 迁移**：breakStorage 的 _migrateFromLocalStorage 模式（读旧→写新→清旧）可用于其他数据迁移
